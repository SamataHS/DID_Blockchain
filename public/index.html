Key Concepts

DID (Decentralized Identifier)
A globally unique ID you create yourself. Format: did:local:uuid. No company or government issues it.
DID Document
A JSON record stored on the ledger containing your public key, name, and creation timestamp.
Public Key
Stored on the ledger. Anyone can use it to verify a message came from you. Safe to share.
Private Key
Kept only by YOU. Never stored on server. Used to prove identity by signing a challenge.

Registration Flow

1
You enter your nameThe server receives your name as the only input needed.
2
Key pair is generatedNode.js crypto module generates a public + private key using Elliptic Curve P-256 algorithm.
3
DID is createdA unique ID is generated: did:local:<uuid>. The DID Document (with your public key) is saved to the ledger.
4
Private key returned onceYour private key is shown to you ONCE and never stored anywhere. You must save it yourself.

Authentication Flow (Passwordless Login)

1
You send your DID + private keyThe server receives your DID to look up your public key on the ledger.
2
Server creates a challengeA random unique string is generated, e.g. did-auth-challenge::did:local:xxx::1710000000. This prevents replay attacks.
3
Challenge is signedYour private key signs the challenge using SHA-256 + ECDSA. Result is a base64 signature string.
4
Signature is verifiedThe server uses your public key (from ledger) to verify the signature. If it matches → you are authenticated!
5
No password ever usedThe private key is only used locally for signing. It is never stored, logged, or transmitted in a way that exposes it.
